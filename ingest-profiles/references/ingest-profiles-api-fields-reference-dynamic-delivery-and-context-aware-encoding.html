    [ 取り込みプロファイル] フィールドのリファレンス

    これは、動的配信（DD）およびコンテキスト認識エンコーディング（CAE）の取り込みプロファイルのフィールドへの参照です。

    こちらもご覧ください取り込みプロファイルAPIフィールドリファレンス（レガシー取り込み）アカウントが従来の取り込みシステム上にある場合。

    プロファイルフィールド

    ユーザーが新しいカスタムプロファイルの基礎として使用する目的でプロファイルをプログラムでプルする場合、ユーザーはプロファイルを削除する必要があります。 id、バージョン、 date_created、 date_last_modified、およびbrightcove_standardフィールド、および名前アカウント内で新しく一意である必要があります。
    トップレベルのプロファイル・オプション
    プロパティ タイプ 説明 DD カエ デフォルト
    id 文字列 プロファイル ID (読み取り専用) なし
    version 整数 プロファイルのバージョン (読み取り専用) なし
    account_id 整数 ビデオクラウドアカウント ID なし
    name 文字列 プロファイル名-アカウント内で一意である必要があります なし
    description 文字列 プロファイルの説明 なし
    date_created 整数 (エポックタイムスタンプ (ミリ秒) プロファイルが作成された日付 (読み取り専用) なし
    date_last_modified 整数 (ミリ秒単位のエポックタイムスタンプ) プロファイルが最後に変更された日付 (読み取り専用) なし
    brightcove_standard ブール値 読み取り専用- falseすべてのカスタムプロファイル用 false
    digital_master オブジェクト デジタルマスターをアーカイブするかどうか、およびその方法を定義します。 {}
    dynamic_origin オブジェクト レンディションとイメージを定義します {}
    renditions 配列 レンディションオブジェクトの配列。 DDまたはCAEには使用されません、これらのプロファイルの空の配列になります []

    デジタルマスターフィールド

    ザ・digital_masterオブジェクトは、後で再トランスコーディングに使用できるデジタルマスターを保存するかどうかとその方法を定義します。見るアーカイブマスター詳細については。

    デジタルマスターフィールド
    プロパティ タイプ 説明 DD カエ デフォルト
    rendition 文字列 アーカイブするレンディション-どちらかpassthroughまたはnone DDおよびCAEプロファイルの唯一のオプションです パススルー
    distribute ブール値 digital_masterをCDNにプッシュして、再生できるようにするかどうか。このオプションは現在、DDまたはCAEプロファイルでは機能せず、に戻ります。falseに設定した場合true false

    動的原点フィールド

    動的原点フィールド
    プロパティ タイプ 説明 DD カエ デフォルト
    renditions 配列 生成されるレンディションの配列。CAE の場合は、必要なオーディオのみのレンディションのみを含めます。を参照してください選択可能なレンディション []
    images 配列 の配列画像ポスターとサムネイルのオブジェクト []
    dynamic_profile_options オブジェクト CAEプロファイルのレンディションオプションを定義します {}

    画像フィールド

    イメージフィールド
    プロパティ タイプ 説明 DD カエ デフォルト
    label 文字列 画像タイプ：いずれかposterまたはthumbnail なし
    height 番号 画像の高さ（ピクセル単位） なし
    width 番号 画像の幅（ピクセル単位） なし

    動的プロファイル・オプションのフィールド

    ダイナミックプロファイルオプションフィールドは、コンテキスト認識エンコーディングを使用する場合のレンディション作成のガイドラインを示します。

    動的プロファイル・オプション
    プロパティ タイプ 説明 DD カエ デフォルト
    video_codecs 配列 混合コーデック出力の場合、レンディションを作成するコーデックの配列。現在、許可されているアイテムの値は次のとおりです。h264そしてhevc [2-1] なし
    video_codec_options 配列 混合コーデック出力の場合、各コーデックのオプションを指定するオブジェクト。各プロパティは、名前の付いたオブジェクトですh264またはhevc、そのプロパティは、そのコーデックのレンディションの制約を設定します。これらのオブジェクトに使用できるプロパティは次のとおりです。
    • min_renditions必須
    • min_bitrate
    • max_bitrate
    • max_first_rendition_bitrate
    • min_resolution
    • max_resolution
    • max_first_rendition_resolution
    注:これらの設定は、グローバルレベルでも使用されます。グローバルレベルとコーデック単位の両方の設定を使用する場合は、値に互換性がある必要があります。たとえば、グローバルレベルで設定された max_renditions は、コーデック単位で設定された最小レンディション値の合計以上である必要があります。
    		 [2-2] なし
    min_renditions 数値 作成するレンディションの最小数 (必須) なし
    max_renditions 数値 作成するレンディションの最大数 (必須) なし
    min_resolution オブジェクト レンディションの最小解像度 なし
    min_resolution.width 番号 レンディションの最小幅 (ピクセル単位) なし
    min_resolution.height 番号 レンディションの最小の高さ (ピクセル単位) なし
    max_resolution オブジェクト レンディションの最大解像度 デフォルトはソース解像度です
    max_resolution.width 番号 レンディションの最大幅 (ピクセル単位) ソース幅
    max_resolution.height 番号 レンディションの最大高さ (ピクセル単位) ソースの高さ
    max_bitrate 番号 レンディションの最大ビットレート (kbps) ソースビットレート

    • 透かしフィールド

    透かし画像は、右下隅の各ビデオフレームに焼き付けられます。

    CAE プロファイルでは、以下のフィールドを使用して、ウォーターマークを指定できます。静的な動的配信インジェストプロファイルに透かしを追加する場合は、Brightcove サポートにお問い合わせください。
    プロパティ タイプ 説明 デフォルト
    url 文字列 画像のURL なし
    width ストリング[3-2] 透かしの幅（ピクセル単位）またはフレーム幅のパーセンテージ-例：20% [3-1] なし
    height ストリング[3-2] 透かしの高さ（ピクセル単位）またはフレームの高さのパーセンテージ-例：10% [3-1] なし
    x ストリング[3-2] ビデオフレームの左端から透かしの中心までの距離 (ピクセルまたはフレーム幅のパーセンテージ) なし
    y ストリング[3-2] ビデオフレームの上端から透かしの中心までの距離 (ピクセルまたはフレーム幅のパーセンテージ) なし

    注

    • [3-1]幅のみを指定すると、画像のアスペクト比が自動的に維持されます
    • [3-2]がwidth、height、x、およびyフィールドは文字列値です。それらには数値（ピクセルまたはパーセンテージ）が含まれています
